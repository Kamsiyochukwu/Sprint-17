Student Dropout Prediction MLOps Pipeline

This project implements a complete MLOps pipeline for predicting student dropout using a machine learning model built on a student performance dataset. The project demonstrates industry-standard machine learning practices by combining DVC for data versioning, MLflow for experiment tracking and model logging, Pytest for automated testing, GitHub Actions for continuous integration and continuous deployment (CI/CD), and Evidently for data drift monitoring. Rather than focusing solely on model performance, this project emphasizes reproducibility, automation, and maintaining a reliable machine learning workflow from development through deployment.

The machine learning pipeline consists of three main stages: data preprocessing, model training, and model evaluation. Running Run.py executes the entire workflow by first preprocessing the data, then training the model in model_train.py, and finally evaluating its performance using evaluation.py. During training, experiments are logged to MLflow, including model parameters, evaluation metrics, and model artifacts, while the dataset is version-controlled with DVC to ensure that every experiment can be reproduced using the same data version. A CI/CD pipeline automates testing and training whenever changes are made to the repository, ensuring that only validated code and models are produced.

To evaluate how well the model performs over time, data drift monitoring was performed using Evidently. The analysis found that Study Hours and Stress Index showed significant drift in Month 2, while Age, Study Hours, Family Income, Attendance, and Department drifted in Month 3. Some of these changes are expected because student populations naturally change over time; for example, students age, family income can fluctuate, and study habits may vary throughout the academic year. However, drift in Attendance is particularly important because it is a strong predictor of student dropout and can have a noticeable impact on model performance. Based on the observed drift, it is recommended that the model be monitored continuously and retrained approximately every two months to maintain prediction accuracy.
